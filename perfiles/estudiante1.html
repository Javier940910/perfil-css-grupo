Especificidad en CSS

Trabajar con CSS sin comprender la especificidad es como escribir reglas en un idioma que no dominas: el navegador siempre decidirá por ti, y casi nunca a tu favor. Aprender que el navegador calcula un "peso" para cada selector (inline > ID > clase > tipo) me cambió la forma de depurar estilos.

La conclusión más importante es que agregar más clases no siempre es la solución; entender el origen del conflicto sí lo es. Cuando los selectores están bien calculados, el CSS se vuelve predecible y fácil de mantener.

Selectores que me costaron entender

  • Selector hijo directo ( > ) Al principio confundía "hijo directo" con "descendiente". div > p solo aplica al <p> inmediato, no a párrafos anidados. Tuve que dibujar el árbol DOM para entenderlo.
  • Pseudo-elemento ::before / ::after No existen en el HTML, son generados por CSS. El truco fue recordar que siempre necesitan content para aparecer, aunque esté vacío.
  • Pseudo-clase :nth-child(n) La fórmula 2n+1 para impares me confundió. Practicar con una lista de 10 ítems cambiando n manualmente fue lo que me aclaró la lógica.
  • Selector de atributo [attr^="valor"] Nunca imaginé que CSS podía "leer" atributos. a[href^="https"] abrió un mundo de posibilidades para estilar sin clases adicionales.

Metodología Mobile-First

Mobile-First no es solo una media query al final del CSS; es una filosofía de diseño. Escribe primero los estilos para la pantalla más pequeña y luego añade complejidad conforme el espacio crece.

El beneficio más concreto es el rendimiento: un móvil solo procesa las reglas base. Las de escritorio ni siquiera se evalúan en pantallas pequeñas.

En el proyecto del perfil lo apliqué así: el contenedor empezó con una sola columna. La media query amplió el layout a dos columnas solo cuando el ancho lo permitía. La lógica fue: ¿qué necesita ver el usuario sí o sí? → eso va en la base. Lo demás se suma después.

/* Base: móvil — una columna */
div.profile-container {
  display: grid;
  grid-template-columns: 1fr;
}

/* Escritorio: dos columnas */
@media (min-width: 768px) {
  div.profile-container {
    grid-template-columns: 260px 1fr;
  }
}
